Sprint-4 (Final Sprint) description:

The final sprint was less rigorous from a deadline perspective than the other 3 sprints, as most of the final tasks were fixing bugs or refactoring error logic based on discussions in class. However, due to the volume of tasks that required completion, this sprint ended with the highest number of completed story points, but over a much longer period of time than the last 3 week-long sprints.

The most complex portion of this sprint was implementing the A/B testing. The dev responsible initially pursued a method using javascript to alternate links to two alternate donate/support pages per event, which resulted in numerous failed tests and code errors. Another dev found the Google Optimize tool within google analytics and passed it off to the previous dev as a potential option. After learning how to enable the tool, the dev implemented Google Optimize to the main branch and commenced the experiment for the alternate donate/support. A more in depth explanation of how this was achieved is below.

We also fixed multiple bugs in this sprint. The largest fix was to the RSVP logic, which originally was managed by a separate controller. We moved the functionality within Event Controller, whenever a POST request from the page is invoked.

We also performed a code review to ensure that the code follows Uber Go style guide. To ensure this, we ran golint and made all changes recommended by the linter. During code review, we found several opportunities to optimize the code further - we removed unnecessary code duplication, modularized the code, and simplified the flow.

A/B Test Result


We used Google Optimize within Google Analytics to conduct the A/B test. Google Optimize was enabled with a javascript header that linked the Optimize experiment with the website. Due to limitations of the program, we only conducted this A/B test on Events 1, 3, and 4, by changing the text from “support” (Original) to “donate” (Variant 1). The experiment tracked the number of users who ended at a donation page linked to each event by alternating the text that users saw with a 50/50 split. This test ran for a total of 8 days with approximately 1200 observations from the various user visits. As of December 13th 2022, event pages with the link text “Donate” got 8.03% conversion, whereas pages with link text “Support” got 3.40% conversion. From this A/B test, we can conclude with a 99% confidence interval that “Donate” is the better choice.
